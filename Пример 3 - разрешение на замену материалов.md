
Загрузка документов **Документ.РазрешениеНаЗаменуМатериалов** из Excel-файла с группировкой строк по номеру документа. Несколько строк Excel с одинаковым номером объединяются в один документ 1С, при этом заполняются три табличные части: основные материалы, аналоги и область действия.

## Общая схема загрузки

```
Excel-файл (17 колонок, строки 2..N)
│
│  Строки группируются по колонке 16 «Номер документа»
│
├── Группа строк «ЦБ-00000002» (4 строки)
│   │
│   ├── Шапка документа
│   │   ├── Номер          ← колонка 16
│   │   ├── Дата           ← алгоритм: ТекущаяДатаСеанса()
│   │   ├── ВариантЗамены  ← предопределённое значение
│   │   ├── Статус         ← предопределённое значение
│   │   └── ...прочие реквизиты...
│   │
│   ├── ТЧ «Материалы» (условие: заполнена кол. 0 ИЛИ 1)
│   │   └── Номенклатура / КоличествоУпаковок / Упаковка
│   │
│   ├── ТЧ «Аналоги» (условие: заполнена кол. 4 ИЛИ 5)
│   │   └── Номенклатура / КоличествоУпаковок / Упаковка
│   │
│   └── ТЧ «ОбластьДействия» (условие: заполнена кол. 15)
│       └── Изделие
│
└── → Документ.РазрешениеНаЗаменуМатериалов в 1С
        (поиск по реквизиту «Номер», режим: обновить существующий)
```

## Выбор типа объекта

В разделе **«Тип объекта»** выберите:

```
Документ → РазрешениеНаЗаменуМатериалов
```

После выбора ObjectFlow загрузит метаданные объекта: реквизиты шапки и три табличные части — **Материалы**, **Аналоги**, **ОбластьДействия**.

## Настройка маппинга

### Шапка документа

| Реквизит | Метод заполнения | Источник / Значение | Тип данных |
|----------|-----------------|---------------------|------------|
| ВариантЗамены | Предопределённое значение | Набор на набор | ПеречислениеСсылка.ВариантЗаменыАналогов |
| ВзаимозаменяемыеАналоги | Из файла | Колонка 8 «Взаимозаменяем» | Булево |
| Дата | Алгоритм | `Результат = ТекущаяДатаСеанса();` | Дата |
| ДатаНачалаДействия | Из файла | Колонка 13 «Дата начала» | Дата |
| ДатаОкончанияДействия | Из файла | Колонка 14 «Дата окончания» | Дата |
| Комментарий | Из файла | Колонка 10 «Комментарий» | Строка |
| Номер | Из файла (поисковый) | Колонка 16 «Номер документа» | Строка(11) |
| Ответственный | Из файла, поиск по Наименованию | Колонка 12 «Ответственный» | СправочникСсылка.Пользователи |
| Статус | Предопределённое значение | Создано | ПеречислениеСсылка.СтатусыРазрешенийНаЗаменуМатериалов |
| УказаниеПоПрименению | Из файла | Колонка 11 «Указание по применению» | Строка |

**Реквизит «Номер» необходимо отметить как поисковый** — именно по нему ObjectFlow будет находить существующий документ для обновления.

**Реквизит «Дата»** заполняется алгоритмом, так как при создании нового документа требуется актуальная дата, а не значение из файла:

```bsl
// Доступные переменные:
// Результат - значение реквизита
// СтрокаДанных - обрабатываемая строка файла
// Объект - обрабатываемый объект
Результат = ТекущаяДатаСеанса();
```

**Реквизит «ВариантЗамены»** — заполнение предопределённым значением перечисления. В поле значения выберите элемент перечисления «Набор на набор».

**Реквизит «Статус»** — аналогично, выберите предопределённое значение «Создано».

**Реквизит «Ответственный»** — поиск элемента справочника «Пользователи» по реквизиту «Наименование», значение берётся из колонки 12.

### ТЧ «Материалы»

**Условие добавления строки:** заполнена колонка 0 (Артикул осн. материала) ИЛИ колонка 1 (Наименование осн. материала) — логика **any**.

| Реквизит ТЧ | Метод заполнения | Источник / Значение | Тип данных |
|-------------|-----------------|---------------------|------------|
| ИдентификаторСтроки | Алгоритм | `Результат = Новый УникальныйИдентификатор;` | УникальныйИдентификатор |
| КоличествоУпаковок | Из файла | Колонка 3 «Кол-во осн.» | Число(15,3) |
| Номенклатура | Из файла, поиск по Наименованию + Артикулу | Колонки 1 и 0 | СправочникСсылка.Номенклатура |
| Упаковка | Из файла, поиск по Коду | Колонка 2 «Код ОКЕИ» | СправочникСсылка.УпаковкиЕдиницыИзмерения |

Для реквизита **«Номенклатура»** настройте поиск по двум свойствам:
- Наименование ← колонка 1
- Артикул ← колонка 0

Для реквизита **«ИдентификаторСтроки»** используется алгоритм:

```bsl
// Доступные переменные:
// Результат - значение строки табличной части
// СтрокаДанных - обрабатываемая строка файла
// Объект - обрабатываемый объект
Результат = Новый УникальныйИдентификатор;
```

### ТЧ «Аналоги»

**Условие добавления строки:** заполнена колонка 4 (Артикул аналога) ИЛИ колонка 5 (Наименование аналога) — логика **any**.

| Реквизит ТЧ | Метод заполнения | Источник / Значение | Тип данных |
|-------------|-----------------|---------------------|------------|
| ИдентификаторСтроки | Алгоритм | `Результат = Новый УникальныйИдентификатор;` | УникальныйИдентификатор |
| КоличествоУпаковок | Из файла | Колонка 7 «Кол-во аналога» | Число(15,3) |
| Номенклатура | Из файла, поиск по Наименованию + Артикулу | Колонки 5 и 4 | СправочникСсылка.Номенклатура |
| Упаковка | Из файла, поиск по Коду | Колонка 6 «Код ОКЕИ аналога» | СправочникСсылка.УпаковкиЕдиницыИзмерения |

Настройка полностью аналогична ТЧ «Материалы», но использует колонки аналогов: наименование из колонки 5, артикул из колонки 4, количество из колонки 7, единица измерения из колонки 6.

### ТЧ «ОбластьДействия»

**Условие добавления строки:** заполнена колонка 15 (Область действия) — логика **any**.

| Реквизит ТЧ | Метод заполнения | Источник / Значение | Тип данных |
|-------------|-----------------|---------------------|------------|
| Изделие | Из файла, поиск по Наименованию | Колонка 15 «Область действия» | СправочникСсылка.Номенклатура |

Для реквизита **«Изделие»** настройте поиск по свойству «Наименование», значение берётся из колонки 15.

## Настройка параметров загрузки

| Параметр | Значение | Описание |
|----------|----------|----------|
| Лист Excel | 0 | Первый лист книги |
| Начальная строка | 2 | Строка 1 содержит заголовки |
| Колонка группировки | 16 «Номер документа» | Строки с одинаковым значением объединяются в один документ |
| Поиск объекта | По реквизиту «Номер» | Документ идентифицируется по номеру из файла |
| Режим обновления | Обновлять существующие (modify) | При нахождении документа — обновить данные |
| Режим записи | Автоматически (auto) | ObjectFlow самостоятельно выполняет запись объекта |
| Обязательные поля | Пропускать строки (skip) | При незаполненных обязательных полях строка пропускается |
| Использовать транзакцию | Нет | Каждый документ записывается независимо |

### Настройки табличных частей

Для каждой из трёх табличных частей в разделе «Настройки ТЧ» задайте:

| Табличная часть | Условие | Колонки условия | Логика | Очищать при заполнении |
|----------------|---------|-----------------|--------|------------------------|
| Материалы | Если заполнены колонки | 0, 1 | Любая (OR) | Да |
| Аналоги | Если заполнены колонки | 4, 5 | Любая (OR) | Да |
| ОбластьДействия | Если заполнены колонки | 15 | Любая (OR) | Да |

Флаг **«Очищать при заполнении»** означает, что перед добавлением новых строк ТЧ будет полностью очищена. Это обеспечивает корректное обновление документа при повторной загрузке.

## Особенности данного примера

**1. Группировка строк по номеру документа.**
Колонка 16 является ключом группировки: все строки с одинаковым значением «Номер документа» объединяются в один объект 1С. Строки с разными номерами формируют отдельные документы. Это позволяет загружать несколько документов из одного файла за один запуск.

**2. Три табличные части с независимыми условиями.**
Каждая ТЧ добавляет строку только при выполнении своего условия. Это позволяет корректно обрабатывать строки, в которых заполнены не все секции: например, 4-я строка группы не имеет значения в «Область действия» и не добавляет запись в ТЧ «ОбластьДействия».

**3. Реквизиты шапки берутся из первой строки группы.**
При группировке ObjectFlow использует значения первой встреченной строки для реквизитов шапки. Если разные строки группы содержат разные значения в колонках шапки, будет использовано значение из первой строки.

**4. Поиск номенклатуры по двум свойствам.**
Для ТЧ «Материалы» и «Аналоги» номенклатура ищется сразу по двум реквизитам: «Наименование» (основной) и «Артикул» (уточняющий). Это повышает точность поиска при наличии номенклатурных позиций с одинаковыми наименованиями.

**5. Генерация уникального идентификатора строки.**
Реквизит «ИдентификаторСтроки» заполняется алгоритмом `Результат = Новый УникальныйИдентификатор;` для каждой строки обеих товарных ТЧ. Это необходимо, если конфигурация 1С требует уникального идентификатора для строк табличных частей.

**6. Дата документа задаётся алгоритмом.**
Реквизит «Дата» намеренно не берётся из файла, а устанавливается в текущую дату сеанса. Это распространённая практика при загрузке документов, когда дата файла неактуальна или не предусмотрена в структуре данных.

**7. Предопределённые значения перечислений.**
Реквизиты «ВариантЗамены» и «Статус» заполняются фиксированными значениями перечислений («Набор на набор» и «Создано» соответственно). Значения задаются по GUID предопределённых элементов, что гарантирует корректное заполнение независимо от языковых настроек базы.

**8. Режим обновления без транзакции.**
Транзакция отключена: каждый документ записывается независимо. Если один документ не прошёл запись (например, из-за ошибки проверки заполнения), остальные документы из файла будут загружены.
