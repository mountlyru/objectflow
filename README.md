## Введение

**ObjectFlow для 1С** — это веб-приложение для автоматизации загрузки данных в 1С:Предприятие. С его помощью вы можете:

- Настроить связи между колонками файла Excel/CSV и реквизитами объектов 1С
- Визуально настроить правила загрузки данных
- Получить готовую обработку для 1С без написания кода
- Экспортировать настройки в удобном формате для документации

### Что нужно для начала работы

- Современный браузер (Chrome, Firefox, Edge, Safari)
- Файл с данными в формате Excel (.xlsx, .xls) или CSV (.csv)
- ZIP-архив с XML-файлами метаданных конфигурации 1С

## Быстрый старт

### Шаг 1: Загрузка файла с данными

1. Откройте приложение в браузере
2. В секции **"Загрузка файла"** нажмите кнопку **"Выбрать файл"** или перетащите файл в область загрузки
3. Поддерживаемые форматы: Excel (.xlsx, .xls), CSV (.csv)
4. После загрузки автоматически отобразится структура файла

## Пошаговые инструкции

### Инструкция 1: Загрузка справочника Номенклатура

#### Подготовка

1. Выгрузите метаданные объекта из 1С (обработка выгрузки метаданных формирует ZIP-архив с XML-файлами)
2. Подготовьте Excel файл с данными номенклатуры

#### Настройка мэппинга

1. **Загрузите файл данных**
   - Выберите Excel файл с номенклатурой
   - Убедитесь, что структура файла отображается корректно

2. **Загрузите метаданные 1С**
   - Выберите ZIP-архив с XML-файлами метаданных
   - Раскройте объект в дереве метаданных

3. **Создайте связи**
   - Кликните на колонку "Наименование" в структуре файла
   - Выберите реквизит "Наименование" из справочника Номенклатура
   - Создайте связь
   - Повторите для других колонок (Артикул, Единица измерения и т.д.)

4. **Настройте поиск для ссылочных полей**
   - Для реквизита "Единица измерения" выберите метод заполнения "Из файла"
   - Укажите реквизит для поиска (например, "Наименование" или "Код")
   - Выберите соответствующую колонку из файла

5. **Настройте параметры загрузки**
   - Укажите лист Excel (если файл содержит несколько листов)
   - Укажите строку начала данных (обычно 2, если первая строка - заголовки)
   - Выберите режим: "Создавать новые объекты" или "Изменять текущие объекты"

6. **Сохраните мэппинг**
   - Нажмите "Сохранить мэппинг" для работы в браузере
   - Или "Экспорт мэппинга" для передачи коллегам

### Инструкция 3: Использование произвольных алгоритмов

#### Произвольный алгоритм в настройках загрузки

1. В настройках загрузки найдите секцию **"Произвольный алгоритм"**
2. В блоке **"Перед записью"** введите код на языке 1С:

   ```1c
       Объект.Дата = ТекущаяДата();
   КонецЕсли;
   ```

<img width="765" height="306" alt="image" src="https://github.com/user-attachments/assets/f1d0e2ed-a04c-40ef-9e53-9abb26426cfe" />

#### Алгоритм как способ заполнения реквизита

Помимо произвольного алгоритма в настройках загрузки, **"Алгоритм"** доступен как отдельный способ заполнения реквизита при создании связи. Это позволяет задать произвольную логику вычисления значения конкретного реквизита.

1. Кликните на реквизит для создания связи
2. Выберите способ заполнения **"Алгоритм"**
3. В редакторе кода введите алгоритм на языке 1С, например:
  ```1c
  // Доступные переменные:
  // Результат - значение реквизита
  // СтрокаДанных - обрабатываемая строка файла
  // Объект - обрабатываемый объект
  Результат = СокрЛП(СтрокаДанных.Колонка1) + " " + СокрЛП(СтрокаДанных.Колонка2);
  ```

> **Когда использовать:** Алгоритм полезен, когда значение реквизита нельзя просто скопировать из файла — например, нужно объединить несколько колонок, выполнить расчёт или преобразование значения.

## Настройки загрузки

### Параметры чтения файла

#### Excel файлы

- **Лист**: Выбор листа из книги Excel (по умолчанию первый лист)
- **Строка начала данных**: Номер строки, с которой начинается чтение данных (заголовки всегда читаются из первой строки)
- **Колонка для группировки строк (идентификатор объекта)**: Позволяет объединить несколько строк файла в один объект 1С. Строки с одинаковым значением в выбранной колонке считаются принадлежащими одному объекту. Используется когда у объекта есть табличные части и данные разбиты по строкам (одна строка = одна строка ТЧ).

> **Пример:** Документ «Поступление товаров» занимает несколько строк Excel — у всех строк одного документа одинаковый номер в колонке «НомерДокумента». Задав эту колонку как колонку группировки, обработка создаст один документ со всеми строками табличной части, а не отдельный документ на каждую строку файла.

<img width="1223" height="139" alt="image" src="https://github.com/user-attachments/assets/b79fb7b7-d93e-4cf6-9a63-284564728a37" />

#### CSV файлы

- **Кодировка**: Автоматическое определение (UTF-8, Windows-1251)
- **Разделитель**: Автоматическое определение (запятая, точка с запятой, табуляция)

### Режимы обновления объектов

- **Создавать новые объекты**: Каждая строка файла создает новый объект
- **Изменять текущие объекты**: Поиск существующего объекта в базе по **сочетанию реквизитов**, которые вы выбираете самостоятельно. Если объект найден — он обновляется, если нет — создаётся новый. Вы можете указать один или несколько реквизитов для поиска, например: Наименование + Владелец, Код + Организация и т.д.

> **Пример:** Для загрузки договоров контрагентов можно настроить поиск по сочетанию реквизитов «Номер» и «Контрагент» — система найдёт существующий договор с таким номером у указанного контрагента и обновит его данные.

### Обязательность реквизита

При создании или редактировании связи можно задать режим обязательности для каждого реквизита. Это радио-группа с тремя вариантами:

- **Не обязательный** — реквизит не проверяется, строка обрабатывается в любом случае
- **Обязательный** — если значение реквизита пустое, срабатывает действие из настройки «Обработка обязательных полей» (пропуск строки или остановка загрузки)
- **Если заполнена колонка** — реквизит становится обязательным только при условии, что указанная колонка файла не пустая. Полезно для опциональных секций данных

В таблице мэппинга обязательность отображается значком: `✓` — всегда обязателен, `если` — условно обязателен, `—` — не обязателен.

### Обработка обязательных полей

- **Пропускать**: Пропустить строку, если не заполнены обязательные поля
- **Прекратить**: Прервать загрузку при первой ошибке

<img width="395" height="169" alt="image" src="https://github.com/user-attachments/assets/0ec88dbe-0ec4-44e3-8b5c-0e1d0f567255" />

### Режим записи объекта

Настройка определяет, каким образом объект записывается в базу данных после заполнения реквизитов:

- **Автоматически** (по умолчанию) — объект всегда записывается автоматически после выполнения всех алгоритмов заполнения. Это стандартный режим для большинства сценариев загрузки.
- **В коде алгоритма** — объект **не записывается** автоматически. Вызов метода `Записать()` должен быть выполнен явно в коде произвольного алгоритма (секция "Перед записью") или в алгоритме заполнения реквизита. Это позволяет реализовать:
  - Условную запись (записывать только при выполнении определённых условий)
  - Пакетную обработку
  - Пропуск записи по условию
  - Дополнительные проверки перед записью

> **Пример использования режима "В коде алгоритма":**
> ```1c
> // Произвольный алгоритм "Перед записью"
> Если НЕ ПустаяСтрока(Объект.Наименование) Тогда
>     Объект.Записать();
> КонецЕсли;
> ```

### Настройки табличных частей

В дереве метаданных рядом с каждой табличной частью отображается кнопка **⚙**. Она открывает боковую панель, где настраивается условие добавления новой строки ТЧ:

- **Всегда** — строка табличной части добавляется для каждой строки файла (или группы строк)
- **Если заполнены колонки** — строка ТЧ добавляется только если указанные колонки файла не пустые. Можно выбрать несколько колонок и задать логику проверки:
  - **И** — все выбранные колонки должны быть заполнены
  - **ИЛИ** — достаточно, чтобы хотя бы одна колонка была заполнена

Настройки задаются **отдельно для каждой табличной части** объекта.

> **Пример:** Документ содержит как основные реквизиты (шапку), так и строки товаров. Если в некоторых строках файла нет данных по товарам (колонки «Номенклатура» и «Количество» пустые), можно настроить условие «Если заполнены колонки» — строка ТЧ «Товары» будет добавляться только когда эти колонки заполнены.

### Методы заполнения реквизитов

#### Из файла
Прямое копирование значения из колонки файла в реквизит объекта. Для ссылочных типов (справочники, документы) выполняется поиск объекта в базе по указанным реквизитам.

**Для ссылочных реквизитов** при выборе метода «Поиск по GUID/наименованию» доступно два режима поиска:
- **Поиск по GUID** — точный поиск по уникальному идентификатору объекта. Используется когда в файле хранится GUID из базы 1С
- **Поиск по наименованию** — поиск объекта по значению реквизита наименование

**Для перечислений** доступен поиск по наименованию значения перечисления.

При настройке поиска ссылочного реквизита в качестве источника значения для каждого реквизита поиска можно выбрать:
- **Колонку файла** — значение берётся напрямую из колонки загружаемого файла
- **Результат другой связи** — значение берётся из результата поиска другого реквизита, связь которого уже настроена в мэппинге

> **Пример:** Загружаем банковские счета. Реквизит «Владелец» (тип СправочникСсылка.Контрагенты) нужно найти по наименованию и владельцу. При этом «Владелец» контрагента — это контрагент, связь которой мы уже настроили ранее. В этом случае при настройке поиска контрагента мы указываем:
> - Номер счета → колонка файла «банковский счет контрагента»
> - Владелец → результат связи «Контрагент» (ранее настроенной)
>
> Таким образом система найдёт банковский счет, где владельцем выступаем конкретный контрагент.

Установка фиксированного значения для всех объектов загрузки. Для ссылочных типов доступен поиск объекта в базе:
- **Поиск по GUID** — введите GUID объекта из базы 1С

#### Значение перечисления
Выбор конкретного значения из перечисления 1С по его наименованию. Используйте этот метод когда нужно зафиксировать одно значение перечисления для всех строк загрузки. Если значение перечисления отличается в зависимости от строки файла — используйте метод «Из файла» (поиск по наименованию).

#### Алгоритм
Произвольный код на языке 1С для вычисления значения реквизита. Используется когда значение нужно рассчитать, преобразовать или собрать из нескольких источников.

**Доступные переменные:**
- `Результат` — переменная, в которую необходимо поместить итоговое значение реквизита
- `СтрокаДанных` — текущая строка обрабатываемого файла (для реквизитов табличных частей)

**Пример:**
```1c
// Объединение значений из двух колонок
Результат = СокрЛП(СтрокаДанных.Колонка1) + " (" + СокрЛП(СтрокаДанных.Колонка2) + ")";
```

## Часто задаваемые вопросы

### Общие вопросы

**Q: Какие форматы файлов поддерживаются?**
A: Поддерживаются Excel (.xlsx, .xls) и CSV (.csv) файлы.

**Q: Какой максимальный размер файла?**
A: Рекомендуемый размер файла до 10 МБ для оптимальной работы. В полученной обработке можно использовать уже файлы любого размера.

**Q: В каком формате загружаются метаданные 1С?**
A: Метаданные загружаются в виде ZIP-архива, содержащего XML-файлы, выгруженные из конфигурации 1С.

### Работа с мэппингом

**Q: Как создать связь для ссылочного поля?**
A: Выберите метод заполнения "Из файла" и укажите реквизиты для поиска (например, Наименование или Код). Для каждого реквизита поиска в качестве источника можно указать колонку файла или результат другой, ранее настроенной связи.

**Q: Что делать, если тип данных не совпадает?**
A: Система покажет предупреждение. В большинстве случаев автоматическое преобразование возможно (например, строка в число). Проверьте данные в файле.

**Q: Можно ли использовать одну колонку для нескольких реквизитов?**
A: Да, можно создать несколько связей с одной колонкой файла.

**Q: Когда использовать способ заполнения "Алгоритм"?**
A: Когда значение реквизита нельзя просто скопировать из файла — например, нужно объединить несколько колонок, выполнить расчёт или условное преобразование значения.

### Настройки загрузки

**Q: Что означает "Строка начала данных"?**
A: Это номер строки, с которой начинается чтение данных из файла. Заголовки всегда читаются из первой строки.

**Q: В чём разница между режимами записи "Автоматически" и "В коде алгоритма"?**
A: В режиме "Автоматически" объект всегда записывается после заполнения реквизитов. В режиме "В коде алгоритма" запись не выполняется — вы должны явно вызвать `Объект.Записать()` в произвольном алгоритме. Это нужно для условной записи или дополнительных проверок.

**Q: Когда использовать произвольные алгоритмы?**
A: Алгоритмы используются для дополнительной обработки данных, например, установки значений по умолчанию, выполнения расчетов или условной записи объекта.

### Экспорт и сохранение

**Q: Где хранятся сохраненные мэппинги?**
A: Мэппинги сохраняются в личном кабинете. Для переноса между устройствами используйте экспорт в JSON.

**Q: Можно ли редактировать экспортированный JSON?**
A: Да, но будьте осторожны - неправильное редактирование может привести к ошибкам при загрузке.

### Генерация обработки

**Q: Нужны ли специальные права для использования обработки в 1С?**
A: Обработка требует стандартных прав на выполнение внешних обработок в 1С.

**Q: Можно ли модифицировать сгенерированную обработку?**
A: Да, обработка содержит стандартный код 1С, который можно редактировать при необходимости.
